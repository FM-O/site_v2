Outils de développement utilisés

Cette page est encore en cours de développement. Par conséquent son design actuel ne représente pas le résultat final. La page est en ligne afin d'aider toute entreprise souhaitant étudier ma candidature à trouver de plus amples informations sur mes compétences.

Environnements de développement

IntelliJ Idea

Niveau estimé :

Détails

IntelliJ est un IDE très puissant développé par JetBrains. Je l'utilise pour la plupart de mes projets et possède une bonne maîtrise de celui-ci. Il permet en effet de gérer de façon simplifiée les projets via plusieurs plugins très utiles. On peut citer notamment l'intégration d'outils de versioning comme Git ou SVN. J'utilise d'autres plugins utiles comme Ant qui me permet de faire de déployer un projet sur un dossier prévu pour une mise en pré-production. J'utilise également YUI Compressor qui permet de minifier les fichiers JS.

J'utilise actuellement Eclipse au sein de SQLI pour le développement lié à mon projet. Je possède donc quelques maîtrises basiques sur cet IDE.

Dreamweaver

Niveau estimé :

Détails

Il s'agit du premier IDE que j'utilisais pour le développement. Je possède donc une bonne maîtrise du logiciel. Cependant j'ai préféré évoluer vers des outils plus puissants et moins orientés WYSIWYG afin de bénéficier de plus de souplesse lors de l'écriture de mon code et surtout dans la gestion de mes différents projets.

Environnements de versioning

Git / Github

Niveau estimé :

Détails

Git est à ce jour le seul outil de versioning que j'utilise. J'en ai une assez bonne maîtrise. il me permet la plupart du temps de gérer de façon plus souple les modifications que j'apporte à mes différents projets et de pouvoir accéder à ceux-ci de n'importe où. La création de différentes branches sur un projet me permet également de prévoir une version de pré-production et une version de production pour chaque projet, bénéficiant ainsi d'un projet toujours fonctionnel en cas d'erreurs sur le développement de la branche "pré-production".
D'autre part, il m'a permis à plusieurs reprise de pouvoir travailler de façon efficace en équipe sur le développement d'applications ou de sites web.
